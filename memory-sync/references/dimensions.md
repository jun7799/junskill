## 1. user_profile — 用户画像

**文件**: `user_profile.md`
**type**: `user`

### 提取要点

从会话中寻找：
- 用户的姓名、昵称、网名、ID
- 职业身份和工作经历（公司名、职位）
- 教育背景
- 设备和系统环境
- 性格特点（从沟通风格、决策方式推断）
- 工作习惯（时段、频率、方式）
- 语言能力
- 个人状态（如失业、找工作等）

### 提取信号

- 文件路径中的用户名（如 `C:\Users\44162`）
- 文件名中的人名（如 `教育部学籍在线验证报告_柯文俊.pdf`）
- 用户明确说的"我是..."、"我在做..."
- 用户提到的工作内容、使用的工具
- 对话风格（语气、用词、决策速度）

## 2. user_projects — 项目全景

**文件**: `user_projects.md`
**type**: `project`

### 提取要点

从会话中寻找：
- 项目名称和描述
- 项目状态（活跃/已完成/归档/进行中）
- 技术栈
- 部署位置和域名
- 客户/对接方
- 自建的 Skills、工具、插件
- 商业布局（卖什么、在哪卖）

### 提取信号

- 项目路径（`D:\归档项目\xxx`）
- 部署 URL 和域名
- "帮我做..."、"继续做..."、"这个项目..."
- 客户名称在目录或对话中出现
- Skill 名称在 `~/.claude/skills/` 目录中

### 状态判断规则

- 有最近30天内的会话 → 活跃
- 有会话但超过30天 → 可能归档
- 有"需求文档"但无实现 → 规划中
- 有部署记录 → 已完成/已部署

## 3. user_tools — 工具链

**文件**: `user_tools.md`
**type**: `reference`

### 提取要点

从会话中寻找：
- 编程语言和框架
- 开发工具（IDE、CLI、平台）
- AI 工具和 API（模型、密钥配置、额度）
- MCP 工具
- 部署平台
- 数据存储（数据库、云存储）
- 第三方服务（支付、通知、邮件等）

### 提取信号

- `import xxx` 语句
- `pip install` / `pnpm add` 命令
- API 密钥和 Base URL（不含实际密钥值）
- 部署命令和平台
- MCP 配置
- "用什么工具..."、"有没有xxx..."

## 4. user_interests — 兴趣方向

**文件**: `user_interests.md`
**type**: `user`

### 提取要点

从会话中寻找：
- 关注的技术领域（按提及频率排序）
- 阅读偏好（书籍、文章、论文类型）
- 关注的人物和博主
- 信息源（网站、公众号、社区）
- 学习方式（看论文、看教程、动手做）
- 工作时段和节奏

### 提取信号

- "我对xxx感兴趣"、"最近在看xxx"
- 分享的文章/论文链接的主题
- 反复提及的技术名词
- 书籍名称在文件路径或对话中出现
- 搜索查询的关键词
- 会话的时间分布模式

## 5. user_pain_points — 痛点

**文件**: `user_pain_points.md`
**type**: `feedback`

### 提取要点

从会话中寻找：
- 技术痛点（反复出现的错误、困难）
- 工作痛点（效率、流程、工具问题）
- 对现有方案的不满
- 失败的尝试
- 资源限制（额度、空间、费用）

### 提取信号

- 错误信息（API Error、timeout、OOM等）
- "不行"、"又出错了"、"还是不行"
- 反复在同一主题上的多轮对话
- "有没有更好的方法"
- "C盘又满了"类表述
- 明确表达的挫败感

### 格式要求

每条痛点必须包含：
1. 问题描述
2. **Why:** 为什么这很重要（从上下文推断）
3. **How to apply:** 未来如何帮助避免

## 合并规则详解

### 新信息（文件中没有的内容）

直接追加到对应章节末尾。

示例：发现用户新项目 "xxx"
→ 在 user_projects.md 的对应表格中添加新行

### 更新的信息（已有但更精确了）

替换旧描述，保留关键词。

示例：旧 "使用Claude Code"，新 "使用Claude Code + 智谱AI API（glm-5.1）"
→ 替换为更精确的版本

### 矛盾的信息

保留新的，旧标注为已更新。

示例：旧 "活跃项目A"，新发现项目A已经3个月没动了
→ 更新状态为"可能归档"，标注"（状态更新于 2026-05-01）"

### 确认的信息（与已有一致）

不动，跳过。不重复记录。

### 新类别

如果发现无法归入现有6个维度的信息：
1. 创建新的记忆文件，遵循 frontmatter 格式
2. 在 MEMORY.md 索引中添加条目
3. 在同步摘要中标注"新增记忆文件：xxx.md"

## 记忆文件格式模板

```markdown
---
name: 记忆名称
description: 一行描述
type: user | feedback | project | reference
---

## 章节标题

内容...
```

**type 取值规则**：
- `user` — 关于用户本人的事实（画像、兴趣、习惯）
- `feedback` — 用户的偏好和痛点（指导行为）
- `project` — 项目信息和状态
- `reference` — 外部资源指针（工具、社交关系、信息源）
