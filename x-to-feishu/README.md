# X to Feishu - X推文采集并上传到飞书多维表

**从Feed流**自动采集符合主题的推文，通过AI生成总结后上传到飞书多维表格。

## ✨ 功能特性

- 🏠 **Feed流采集**：从个人主页时间线滚动采集（更自然、更安全）
- 🎯 **主题订阅**：支持多个主题，每个主题对应一组关键词
- ⏰ **时间过滤**：只采集指定天数内的新鲜内容
- 📊 **热度筛选**：按观看数、点赞数、转发数筛选
- 🤖 **AI总结**：自动生成推文主题、提取关键词、生成梗概
- 📤 **飞书上传**：直接上传到飞书多维表，无需手动整理
- 🛡️ **安全增强**：随机延迟、保守策略，降低封号风险

## 🎯 适用场景

- 📰 **行业资讯采集**：采集AI、科技、金融等领域的最新动态
- 🔥 **热点追踪**：快速发现某个话题的热门讨论
- 📈 **竞品分析**：监控竞品或KOL的推文动态
- 📝 **内容策划**：收集灵感，了解热门内容形式
- 🎓 **研究调研**：采集特定主题的社交媒体数据

## 🚀 快速开始

### 首次使用：配置Cookie

**第一步：获取Cookie**

在浏览器中登录 https://x.com 后，使用以下任一方法获取Cookie：

**方法1（推荐）：**
1. 按 `F12` 打开开发者工具
2. 切换到 `Console` 标签
3. 输入并回车：`copy(document.cookie)`
4. Cookie已复制到剪贴板

**方法2：**
1. 按 `F12` 打开开发者工具
2. 切换到 `Application` (或 `存储`) 标签
3. 左侧选择 `Cookies` -> `https://x.com`
4. 手动复制所有Cookie（格式：`name1=value1; name2=value2; ...`）

**第二步：配置Cookie文件**

将获取的Cookie粘贴到文件中：
```
.claude/skills/x-to-feishu/cookie.txt
```

直接粘贴Cookie内容即可，无需其他格式。示例：
```
guest_id=v1%3A123456789; auth_token=abc123def456; ct0=xyz789...
```

### 使用方法

配置好Cookie后，直接在Claude Code中告诉我你的需求：

```
帮我采集AI相关的推文
```

我会使用默认配置（10条、3天内、100+观看），你也可以自定义：

```
帮我采集20条AI推文，要最近7天的，观看数超过1000
```

## 📖 详细说明

### 主题配置

在 `config.json` 中配置主题关键词：

```json
{
  "themes": {
    "AI": {
      "keywords": [
        "gpt", "chatgpt", "claude", "gemini",
        "ai", "llm", "openai", "anthropic"
      ],
      "description": "AI人工智能相关"
    },
    "区块链": {
      "keywords": [
        "bitcoin", "ethereum", "crypto",
        "nft", "web3", "defi", "区块链"
      ],
      "description": "区块链加密货币相关"
    }
  }
}
```

添加新主题只需在 `themes` 中添加即可。

### 筛选逻辑

推文必须同时满足：
1. ✅ 内容包含至少一个主题关键词
2. ✅ 发布时间在指定范围内
3. ✅ 观看数达到最小阈值

### 采集流程

1. **打开X首页** - 访问个人Feed流
2. **设置Cookie** - 模拟登录状态
3. **滚动采集** - 自动滚动加载更多推文（带随机延迟）
4. **筛选推文** - 实时过滤符合条件的推文
5. **AI总结** - 为每条推文生成主题、关键词、梗概
6. **飞书上传统** - 上传到飞书多维表

整个过程大约需要 **2-5分钟**（取决于网络速度和采集数量）。

### 飞书多维表字段

| 字段名 | 说明 | 数据来源 |
|--------|------|---------|
| 推文主题 | AI生成的推文主题 | Claude AI总结 |
| 推文链接 | 原帖链接 | 推文URL |
| 发布时间 | 作者发布时间 | 推文时间戳 |
| 关键词 | AI提取的3个关键词 | Claude AI总结 |
| 推文梗概 | AI生成的推文概要 | Claude AI总结 |
| 作者 | 发布者用户名 | @username |
| 点赞数 | 点赞数量 | 推文互动数据 |
| 评论数 | 评论数量 | 推文互动数据 |
| 转发量 | 转发数量 | 推文互动数据 |
| 阅读量 | 观看数量 | 推文统计数据 |

## ⚙️ 配置选项

| 参数 | 默认值 | 说明 |
|------|--------|------|

| 时间范围 | 3天 | 只采集N天内的推文 |

## 🛡️ 安全使用建议（重要！）

### ⚠️ 封号风险评估

| 使用频率 | 封号风险 | 建议 |
|---------|---------|------|
| 1-2次/天 | 🟢 低风险 | 可以正常使用 |
| 3-5次/天 | 🟡 中风险 | 建议降低频率 |
| 5+次/天 | 🔴 高风险 | **强烈不推荐** |

### 🛡️ 安全策略

#### 1️⃣ **使用测试账号**
```
⚠️ 不要用主账号！
✅ 建议注册一个专门的测试账号用于采集
```

#### 2️⃣ **控制使用频率**
- 每天使用不超过 2-3 次
- 每次间隔至少 1 小时
- 避免在短时间内重复采集同一主题

#### 3️⃣ **保守配置**
- 默认采集 10 条即可，不要贪多
- 观看数阈值设为 100+，过滤掉低质量推文
- 时间范围设为 3-7 天，不要太宽

#### 4️⃣ **随机延迟机制**
- 每次滚动延迟 2-5 秒（随机）
- 避免固定的行为模式
- 更接近真实用户操作

#### 5️⃣ **Feed流模式优势**
- 模拟真实用户浏览首页的行为
- 看到的是关注账号的内容，质量更高
- 比搜索更不容易被检测为爬虫

### 📌 安全检查清单

使用前请确认：
- [ ] 使用的是测试账号（非主账号）
- [ ] 今天使用次数不超过 2-3 次
- [ ] 与上次使用间隔至少 1 小时

## 🐛 故障排查

### 问题1：Cookie无效

**症状：**
```
❌ 无法加载搜索结果，显示登录页面
```

**解决方法：**
1. 重新登录X网站
2. 清除浏览器缓存后再次登录
3. 复制最新的Cookie（确保完整）
4. 尝试使用隐身模式登录并获取Cookie

### 问题2：采集不到推文

**症状：**
```
✅ 采集完成，但只找到1-2条推文
```

**可能原因和解决：**
- **关键词太严格**：增加更多相关关键词
- **时间范围太短**：将3天改为7天或更长
- **观看数阈值太高**：将100改为10或0
- **搜索结果不足**：尝试其他关键词

### 问题3：飞书上传承败

**症状：**
```
❌ 上传失败：table not found
```

**解决方法：**
1. 检查 config.json 中的飞书配置是否正确
2. 确认 Base ID 和 Table ID 是否匹配
3. 检查飞书应用权限是否开通
4. 确认网络连接正常

### 问题4：AI总结失败

**症状：**
```
❌ AI API调用失败
```

**解决方法：**
1. 检查AI API密钥是否配置
2. 确认API额度是否充足
3. 检查网络连接
4. 可以跳过AI总结，直接上传原始数据
