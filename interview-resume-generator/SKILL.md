# 访谈式简历生成器

通过智能访谈引导用户生成专业简历，或基于现有简历进行智能优化。支持 Markdown 和 HTML 两种输出格式。

## 触发场景

### 路径A：从零生成（访谈式）
- 用户说"帮我写简历"、"生成简历"、"做一份简历"
- 用户说"我要找工作，需要简历"
- 用户说"更新一下我的简历"
- 用户说"准备面试，需要简历"

### 路径B：简历优化（已有简历）
- 用户说"帮我优化简历"、"看看我的简历"、"改进简历"
- 用户说"简历诊断"、"我有简历想改改"、"优化一下这份简历"
- 用户直接粘贴或上传了一份简历内容

### 通用触发
- 使用 `/interview-resume-generator` 命令

## 核心流程

```
入口分流
├── 路径A: 从零生成
│   阶段0 → 阶段0.5 → 阶段1 → ... → 阶段6 → 生成MD → 阶段7: HTML（可选）
│
└── 路径B: 简历优化
    阶段B0 → 阶段B1 → 阶段B2 → 阶段B3（分析报告）→ 用户确认 → 阶段B4（生成MD + HTML）
```

---

## 入口分流

**判断逻辑**：如果用户已提供简历内容（粘贴了文字、上传了文件、说"优化简历"等），直接进入**路径B**。否则确认路径：

```
问题: "你好！我是简历小助手。请问你需要哪种服务？"

选项:
1. label: "从零开始，帮我写简历"
   description: "通过访谈问答，一步步生成全新简历"

2. label: "我已有简历，帮我优化"
   description: "提供现有简历，AI 分析后优化改进"
```

## 行业定制化追问策略

根据 `industry_type` 在工作经历和项目经历阶段使用不同的追问策略：

### 技术研发类（tech）

**核心关注点**：技术深度、架构能力、性能优化、技术难点攻克

**工作经历追问**：
```
问题: "在这个岗位上，你遇到的最大技术挑战是什么？如何解决的？"
```

```
问题: "请说说你使用的主要技术栈，以及你在技术选型上的考量"
```

**项目经历追问**：
```
问题: "这个项目的技术架构是怎样的？你为什么选择这个架构？"
```

```
问题: "项目中有哪些性能优化或技术亮点？\n\n提示：\n- 性能提升（响应时间、吞吐量）\n- 稳定性改进（可用性、容错）\n- 技术创新（新方案、开源贡献）"
```

**成果量化提示**：
- 性能提升：响应时间、QPS、并发量
- 稳定性：可用性99.9%、故障率降低
- 代码质量：覆盖率、Bug率
- 工程效率：构建时间、部署频率

---

### 产品类（product）

**核心关注点**：用户洞察、数据驱动、产品规划、跨部门协作

**工作经历追问**：
```

```

```

**项目经历追问**：

```

**成果量化提示**：
- 用户指标：DAU/MAU、留存率、转化率
- 业务指标：GMV、营收、ARPU
- 产品指标：功能使用率、NPS、满意度
- 迭代效率：需求交付周期、迭代频率

### 7.2 生成HTML文件

**如果用户选择需要，则生成以下文件结构：**

```
{工作目录}/
├── {姓名}_简历_{日期}.md          # 已生成的Markdown文件
├── {姓名}_简历.html                # 主HTML文件（现代简约风格）
└── templates/                      # 模板目录
    ├── resume_modern.html          # 现代简约
    ├── resume_classic.html         # 经典传统
    ├── resume_creative.html        # 创意设计
    └── resume_tech.html            # 科技暗黑
```

### 7.3 HTML模板说明

**4种模板风格（基于 Claude Design 方法论）：**

| 模板 | 文件名 | 特点 | 适用场景 |
|------|--------|------|----------|
| 现代简约 | resume_modern.html | 双栏布局，排版驱动，oklch蓝 | 互联网、科技公司 |
| 经典传统 | resume_classic.html | 衬线字体，横线分隔，深色调 | 国企、金融、政府 |
| 创意设计 | resume_creative.html | 非对称排版，强烈字重对比 | 设计、创意、广告 |
| 科技暗黑 | resume_tech.html | 等宽字体，终端风格，深色底 | 程序员、技术岗 |

### 7.4 HTML功能特性

每个HTML模板都包含：

1. **模板切换**：点击"切换模板"按钮，在4种风格间自由切换
2. **导出PDF**：点击"导出PDF"按钮，调用浏览器打印功能
3. **导出PNG**：点击"导出PNG"按钮，使用html2canvas生成高清图片
4. **响应式设计**：适配手机、平板、电脑等不同设备
5. **打印优化**：打印时自动隐藏工具栏，优化输出效果
6. **Tweaks调参**：通过CSS变量暴露设计参数（颜色/字号/间距），支持实时调整

### 7.5 设计原则

所有模板基于 Claude Design 方法论：

- **反AI味**：无大面积渐变、无emoji装饰、无圆角+左边框套路
- **oklch配色**：色彩通过oklch色彩空间定义，确保和谐统一
- **排版驱动**：用字号/字重/间距建立视觉层次，不依赖颜色和装饰
- **现代CSS**：CSS Grid布局、text-wrap: pretty、CSS自定义属性

### 7.5 HTML生成提示词

生成HTML时，使用以下结构：

**主HTML文件（{姓名}_简历.html）**：
- 使用现代简约风格（modern）作为默认风格
- 模板路径指向 `./templates/`
- 包含完整的简历数据

**templates目录下的模板**：
- 每个模板都包含相同的简历数据
- 仅样式不同，内容一致
- 模板间路径使用 `./` 相对路径

**设计要求**：
- 颜色使用 oklch 色彩空间定义
- 布局使用 CSS Grid
- 禁止大面积渐变、emoji装饰、圆角+左边框套路
- 通过 CSS 自定义属性暴露设计参数（Tweaks）

## 阶段B3: AI 分析报告

**目标**：对现有简历进行全面审视，输出结构化分析报告

### 分析维度

#### 1. 结构完整性检查

| 板块 | 检查项 |
|------|--------|
| 基本信息 | 是否有姓名、联系方式、求职意向 |
| 教育背景 | 是否有学校、专业、学历、时间 |
| 工作经历 | 是否有公司、职位、时间、职责描述 |
| 项目经历 | 是否有项目名、角色、背景、成果 |
| 专业技能 | 是否有技能分类和熟练程度 |
| 自我评价 | （可选）是否有个人特点描述 |

#### 2. 内容质量评估（STAR法则）

使用 `references/quality_indicators.md` 的评分标准：

| 维度 | 分值 | 判断标准 |
|-----|------|---------|
| 具体性 | 0-3分 | 是否有具体的场景、角色、任务描述 |
| 量化程度 | 0-3分 | 是否有数字、比例、具体成果 |
| 完整性 | 0-2分 | 是否回答了问题的核心要点 |
| 亮点度 | 0-2分 | 是否有独特价值、突出成就 |

根据场景类型调整权重（社招重量化、校招重具体性、转行重完整性）。

#### 3. 行业适配性分析

根据 `industry_type` 检查：
- 技能关键词是否匹配目标岗位
- 经历描述是否符合行业标准（如技术岗是否提到技术栈、架构）
- 是否缺少该行业关键信息（参考各行业追问策略中的"成果量化提示"）

#### 4. 具体优化建议

逐板块列出改进点：
- 哪些描述太笼统，需要具体化
- 哪些成果缺少量化数据，建议补充什么类型的数字
- 哪些内容冗余或与目标岗位无关，建议删除/精简
- 哪些板块缺失，建议补充

## 阶段B4: 生成优化简历

**目标**：基于分析结果，生成优化后的简历

## 回答质量评估机制

### 评分维度

| 维度 | 分值 | 判断标准 |
|-----|------|---------|
| 具体性 | 0-3分 | 是否有具体的场景、角色、任务描述 |
| 量化程度 | 0-3分 | 是否有数字、比例、具体成果 |
| 完整性 | 0-2分 | 是否回答了问题的核心要点 |
| 亮点度 | 0-2分 | 是否有独特价值、突出成就 |

### 响应策略

| 总分 | 等级 | 行为 |
|-----|------|-----|
| 0-3 | 低 | 必须追问，提供示例引导 |
| 4-6 | 中 | 适度追问，挖掘更多细节 |
| 7-8 | 良 | 简单确认，记录亮点 |
| 9-10 | 优 | 深度挖掘，成为简历核心卖点 |

### 质量判断示例

**低质量回答**：
- "我负责开发" → 太笼统，必须追问
- "做了很多项目" → 没有具体内容，必须追问
- "工作很努力" → 没有实际成果，必须追问

**高质量回答**：
- "负责抖音电商首页，使用React+TS重构，将首屏加载时间从3s优化到1.2s" → 具体且有量化
- "带领8人团队完成支付系统重构，日处理订单从10万提升到100万" → 有角色、有成果

## 注意事项

1. **渐进式提问**：一次只问一个问题，不要一次问太多

3. **提供示例**：每个问题都提供填写示例
4. **允许跳过**：用户可以跳过不合适的问题
5. **智能追问**：根据回答质量决定是否追问
